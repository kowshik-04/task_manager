## 📋 Pre-Deployment Checklist

### Security
- [ ] JWT_SECRET is **minimum 32 random characters**
- [ ] DATABASE_URL uses strong password (auto-generated by Railway)
- [ ] FRONTEND_URL is HTTPS only
- [ ] All secrets use environment variables (never hardcoded)
- [ ] Rate limiting enabled (200 req/15min)
- [ ] CORS configured to frontend domain only

### Configuration
- [ ] NODE_ENV=production (not development)
- [ ] JWT_EXPIRES_IN=1d or appropriate expiry
- [ ] Database backup strategy planned
- [ ] Error monitoring set up (Sentry, LogRocket, etc.)
- [ ] Backend logs accessible (Railway dashboard)

### Data
- [ ] Database schema verified (`prisma db push` successful)
- [ ] Demo seed data created or real data migrated
- [ ] Backup of existing data taken (if migrating)
- [ ] User accounts created for admins and testers

## 🚀 Railway Deployment (Recommended)

Railway is the simplest production deployment for this stack.

### Step 1: Setup Railway Account

```bash
# Create account at railway.app
# Create new project

# Install CLI
npm i -g @railway/cli

# Login
railway login

# Initialize project
railway init
```

### Step 2: Add PostgreSQL

**Via Railway Dashboard**:
1. Click "New Service"
2. Select "PostgreSQL"
3. Click "Provision"

Railway automatically:
- Creates database named `railway`
- Sets DATABASE_URL environment variable
- Enables private networking
- Configures automatic backups

**Via CLI**:
```bash
# View available plugins
railway service

# Add PostgreSQL
railway service add -s postgres
```

### Step 3: Deploy Backend

```bash
# Add backend service
railway add

# Select Node.js from list
# Specify: ./backend directory
```

**Configure Backend Service**:

Go to Railway Dashboard → Backend Service → Variables

Add environment variables:
```env
NODE_ENV=production
JWT_SECRET=generate_32_random_characters_here
JWT_EXPIRES_IN=1d
FRONTEND_URL=https://your-frontend-domain.vercel.app

# DATABASE_URL is auto-set by Railway PostgreSQL plugin
# PORT is auto-set by Railway (5000-5999)
```

**Set Build & Start Commands**:

In Railway Dashboard → Backend → Settings → Build:

**Build Command**:
```bash
npm install && npx prisma generate
```

**Start Command**:
```bash
npx prisma db push --skip-generate && npx prisma db seed && npm start
```

Railway will:
1. Install dependencies
2. Generate Prisma client
3. Sync database schema
4. Seed demo data
5. Start backend server
6. Run healthcheck (GET /health)

### Step 4: Deploy Frontend

**Option A: Vercel (Recommended - Free)**

1. Push frontend code to GitHub
2. Go to vercel.com → Import Project
3. Select frontend repository
4. Configure:
   - Build Command: `npm install && npm run build`
   - Output Directory: `dist`
   - Environment Variable: `VITE_API_URL=https://ttm-api.up.railway.app`
5. Click "Deploy"
6. Vercel auto-redeploys on git push

**Option B: Railway Static**

```bash
# Build frontend
npm run build

# Deploy dist folder
railway add
# Select "Static"
# Point to ./frontend/dist directory
```

### Step 5: Connect Services

**Update Frontend Environment**:

In Vercel Dashboard or `frontend/.env`:
```env
VITE_API_URL=https://your-backend.up.railway.app
```

Redeploy frontend (push to GitHub or manual in Vercel).

## 🐛 Troubleshooting Deployment

### Backend won't start

**Check logs**:
```bash
railway logs -s backend
```

**Common errors**:

1. **"Failed to connect to database"**
   - Wait 60 seconds (PostgreSQL startup)
   - Verify DATABASE_URL is set
   - Restart: `railway service restart -s backend`

2. **"ENOTFOUND prisma"**
   - Missing `npx prisma generate` in build
   - Rebuild: `railway redeploy -s backend`

3. **"JWT_SECRET not found"**
   - Set JWT_SECRET environment variable
   - Restart backend

### Frontend can't reach backend

1. **Check CORS**:
   - Backend must have FRONTEND_URL set
   - Verify in logs

2. **Test health endpoint**:
   ```bash
   curl https://your-backend.up.railway.app/health
   ```

3. **Check frontend environment**:
   ```
   VITE_API_URL=https://your-backend.up.railway.app
   ```

4. **Browser console**:
   - Check for CORS errors
   - Look for failed requests to backend

### Database issues

1. **"Relation does not exist"**
   - Schema not pushed: `railway exec -s backend npx prisma db push`

2. **"No seed data"**
   - Run seed: `railway exec -s backend npx prisma db seed`

3. **"Database locked"**
   - Restart PostgreSQL: `railway service restart -s postgres`

## 🔐 Security in Production

### Secrets Management

**Never commit secrets**. Use environment variables only:

```bash
# Bad (do NOT do this)
const JWT_SECRET = "hardcoded_secret"; // ❌

# Good (Railway environment variable)
const JWT_SECRET = process.env.JWT_SECRET; // ✓
```

**Railway secrets**:
- Set in Dashboard → Service → Variables
- Not visible in public URLs or logs
- Rotatable anytime

### Network Security

- **HTTPS only**: All traffic encrypted
- **Private DB**: PostgreSQL not exposed to internet
- **Rate limiting**: 200 req/15min per IP
- **CORS**: Only frontend domain allowed
- **Headers**: Helmet enforces security headers

### Database Security

- **Passwords**: Auto-generated by Railway (strong)
- **Backups**: Railway automatic daily backups
- **Access**: Private network only (not public internet)
- **Encryption**: Encrypted at rest and in transit

## 🔄 Deployment Workflow

### Making Changes

1. **Local development**:
   ```bash
   npm run dev  # frontend
   npm run dev  # backend
   ```

2. **Testing**:
   ```bash
   npm run build  # verify builds
   npm test       # run tests (if added)
   ```

3. **Push to GitHub**:
   ```bash
   git add .
   git commit -m "feature: ..."
   git push origin main
   ```

4. **Auto-deploy**:
   - Railway: Auto-redeploys on push to main
   - Vercel: Auto-redeploys on push to main

### Rolling Back

```bash
# If deployment fails:

# Revert code
git revert <commit_hash>
git push

# Or manually in Railway Dashboard:
# Select previous deployment → "Revert"
```
